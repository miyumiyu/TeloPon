# 🎛️ Release V1.22b: New Auto Segment Mode & UI Overhaul!

> 👉 **[Download TeloPon v1.22b](https://github.com/miyumiyu/TeloPon/releases/tag/v1.22b)**

The highlight of this update is the brand-new **"Auto Segment"** feature.
The talk mode selection UI has also been redesigned for a more intuitive experience.

---

## ✨ What's New & Improved

### 🆕 New Feature: Auto Segment Mode

When a set amount of time has passed since the last telop was displayed, the AI automatically outputs a telop summarizing and organizing the topics covered so far.

* Select **"Auto Segment"** from the "Talk Mode" combo box in the UI to enable
* Use the "Intervention Time (seconds)" slider to set the trigger interval
* A **live countdown** to the next trigger is displayed next to the combo box

**Perfect for:**
News/information streams or long chat sessions where periodic topic recaps help viewers keep up with the conversation. An interval of 60–120 seconds is recommended.

---

### 🔄 UI Improvement: Talk Mode Combo Box

The previous checkbox-based "Auto Talk ON/OFF" has been replaced with a **combo box for selecting talk mode**.

| Mode | Behavior |
|---|---|
| **None** | No automatic intervention |
| **Auto Talk** | AI proactively introduces a topic after N seconds of silence (existing feature) |
| **Auto Segment** | AI summarizes and organizes topics N seconds after the last telop (new feature) |

* The countdown next to the combo box shows **seconds until the next trigger** at a glance
* The countdown keeps running even while the AI is thinking, so you always know the exact trigger timing
* Mode can be switched even during a live session

---

### 🔧 Feature Improvement

* YouTube & Twitch plugins can now be connected and disconnected after a live session has started

---

### 🐛 Bug Fixes & Stability

* Fixed an issue where brief noises (coughs, ambient sounds) were incorrectly detected as speech, causing frequent empty turns
  (Audio must now last at least 0.3 seconds to be treated as speech)
* Fixed an issue where the AI incorrectly recognized the streamer as silent even while speaking
* Fixed an issue where the API connection from the server was not correctly re-established after disconnection
