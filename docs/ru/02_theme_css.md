# TeloPon: Руководство по настройке тем и CSS 🎨

В TeloPon вы можете свободно настраивать дизайн телопов, отображаемых в OBS, с помощью «Тем».

Система рендеринга TeloPon высокоэффективна — она использует архитектуру **«общий HTML-скелет с заменяемыми индивидуальными CSS-дизайнами»**. Это означает, что для создания новой темы **достаточно создать один CSS-файл**.

---

## 📁 1. Где сохранять файлы тем

Чтобы создать новую тему, создайте и сохраните CSS-файл в папке `TeloPon/themes/`.

```text
TeloPon/
 └── themes/
      ├── default.css        ← Стандартная тема
      └── my_custom.css      ← 🌟 Просто добавьте сюда свой CSS-файл!
```
※ Любой добавленный CSS-файл автоматически появится в выпадающем списке «⚙️ Тема телопа» в интерфейсе TeloPon.

---

## 🏗️ 2. [Важно] Структура HTML и два контейнера телопов

Дисплей TeloPon (`base.html`) имеет два встроенных контейнера телопов: **«Обычный контейнер»** и **«Специальный контейнер»**.
Какой контейнер используется, определяется автоматически по «ID окна» (описано ниже), который задаёт ИИ.

### ① Обычный контейнер телопа (normal group)
Стандартный контейнер телопа, отображающийся последовательно, как правило, в верхнем правом углу экрана.
```html
<div id="teloop-container" class="ID окна и ID макета здесь">
    <div id="teloop-badge"></div>
    <div id="text-wrapper">
        <div class="topic-line">Тема</div>
        <div class="main-line">Основной текст</div>
    </div>
</div>
```

### ② Специальный контейнер (special group)
Используется, когда нужно отобразить телоп «независимо (перекрывая)» с обычным телопом — например, пояснительные телопы внизу экрана или представления имён.
```html
<div id="explain-container" class="ID окна здесь">
    <div id="exp-badge"></div>
    <div id="exp-text-wrapper">
        <div class="topic-line">Тема</div>
        <div class="main-line">Основной текст</div>
    </div>
</div>
```

## 🪪 3. Доступные ID окон и управление системой

TeloPon инструктирует ИИ использовать предопределённые «ID окон». В CSS вы пишете правила дизайна и системные команды, нацеленные на класс (`.window-xxx`) для каждого ID окна.

### 📌 ID обычных контейнеров (normal group)
Применяются к `#teloop-container` и исчезают по истечении времени, установленного в интерфейсе в разделе «Длительность обычного отображения (сек)».

* **`.window-simple`**: Для наиболее стандартных, простых телопов.
* **`.window-variety`**: Для броских дизайнов в стиле развлекательного шоу.
* **`.window-news`**: Для официальных дизайнов в стиле новостной трансляции.
* **`.window-cute`**: Для рукописных или милых/kawaii дизайнов.
* **`.window-digital`**: Для дизайнов в стиле киберпанк или цифровой эстетики.
* **`.window-reply`**: Для ответов зрителям или упоминаний (например, «Эй, @имя_пользователя!») — широкоформатные дизайны, привлекающие внимание.

### 📌 ID специальных контейнеров (special group)
Применяются к `#explain-container`. Чтобы сообщить системе «это специальный контейнер», необходимо **указать `--window-group: special;` в CSS**.

* **`.window-explain`**: Для длинных пояснений или дополнительной информации, отображаемых заметно внизу экрана.
* **`.window-nameplate`**: Для отображения «псевдонима + имени» в телевизионном стиле крупным текстом.

## 🔊 4. Воспроизведение звуков и управление системой (CSS-переменные)

Одна из главных особенностей TeloPon — возможность **«передавать настройки обратно в систему (на сторону Python/JS) из CSS»**. Это делается с помощью «CSS-переменных (деклараций, начинающихся с `--`)».

### 💡 Воспроизведение звуковых эффектов (`--sound-file`)
Чтобы воспроизвести звуковой эффект при появлении телопа, запишите `--sound-file` в CSS. Укажите аудиофайл из папки `TeloPon/sounds/`.

```css
/* Пример: воспроизводить звук "pop" при появлении окна variety */
.window-variety {
    background: linear-gradient(135deg, #ff00cc, #333399);
    color: white;
    /* Указывает системе воспроизвести TeloPon/sounds/sound1.mp3 */
    --sound-file: "sound1.mp3";
}
```

### 💡 Переключение контейнера в «специальный» режим (`--window-group`)
Как упоминалось выше, при создании дизайнов специальных контейнеров, таких как `window-explain`, пропуск этой переменной приведёт к тому, что они будут обработаны как обычный контейнер.

```css
.window-explain {
    position: absolute;
    bottom: 40px;
    /* Включение этого приводит к рендерингу в "#explain-container" */
    --window-group: special;
    --sound-file: "sound2.mp3";
}
```

### 💡 Принудительное переопределение длительности отображения (`--display-duration`)
Можно принудительно установить фиксированную длительность отображения для конкретного телопа, игнорируя секунды, настроенные в интерфейсе (например, 5 секунд).

```css
.window-nameplate {
    /* Переопределить настройку интерфейса — принудительно скрыть табличку с именем ровно через 5 секунд */
    --window-group: special;
    --display-duration: 5;
}
```

## 📐 5. ID макетов (настройка баланса текста)

В дополнение к ID окна, обычный контейнер (`#teloop-container`) также получает «ID макета». Используйте его для настройки **соотношения размеров темы и основного текста**.

* **`.layout-flat`**: Плоский макет — тема и основной текст одного размера.
* **`.layout-big-top`**: Крупный текст в верхней строке (тема), меньший в нижней (основной).
* **`.layout-big-bottom`**: Меньший текст в верхней строке (тема), крупный и заметный в нижней (основной).

**Пример использования CSS:**
```css
/* Когда задан flat, можно применять исключения, например убрать рамки для вида только текста */
.window-variety.layout-flat .topic-line {
    background: none !important;
    font-size: 36px !important;
}
```

## 🎬 6. Реализация анимаций (`.visible`)

Когда телоп появляется на экране, система добавляет класс **`.visible`** к контейнеру.
Используйте это вместе с CSS `transition` и `transform` для создания плавных анимаций появления.

```css
/* =========================================
   Обычный телоп (увеличивается и появляется с затуханием из верхнего правого угла)
   ========================================= */
#teloop-container {
    position: absolute; right: 30px; top: 60px;
    opacity: 0;
    transform: scale(0.8);
    transition: all 0.5s;
}
#teloop-container.visible {
    opacity: 1;
    transform: scale(1);
}

/* =========================================
   Пояснительный телоп (выезжает снизу вверх)
   ========================================= */
.window-explain {
    position: absolute; left: 40px; bottom: 40px;
    opacity: 0;
    transform: translateY(30px);
    transition: all 0.6s cubic-bezier(0.2, 0.8, 0.2, 1);
    --window-group: special;
}
.window-explain.visible {
    opacity: 1;
    transform: translateY(0);
}
```

## 💡 7. Советы по разработке (отладка)

Корректировка CSS темы путём каждый раз ожидания, пока ИИ заговорит, занимает много времени.
Когда нужно проверить и настроить дизайн, запустите TeloPon в режиме отладки (`-d`).

```bash
python telopon_live.py -d
```

В нижней части интерфейса появится **«Панель ручного управления»**.
Введите тему, основной текст и ID окна, которое хотите протестировать, затем нажмите кнопку «Принудительное отображение», чтобы запускать тесты рендеринга (включая подтверждение воспроизведения звука) сколько угодно раз — без ИИ.
Доводите дизайн до совершенства, наблюдая в реальном времени за применением изменений CSS в браузерном источнике OBS!
