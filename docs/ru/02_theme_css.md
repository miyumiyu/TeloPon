# TeloPon: Руководство по настройке тем и CSS 🎨

В TeloPon вы можете свободно настраивать дизайн телопов, отображаемых в OBS, с помощью «Тем».

Система рендеринга TeloPon высокоэффективна — она использует архитектуру **«общий HTML-скелет с заменяемыми индивидуальными CSS-дизайнами»**. Это означает, что для создания новой темы **достаточно создать один CSS-файл**.

---

## 📁 1. Где сохранять файлы тем

Чтобы создать новую тему, создайте и сохраните CSS-файл в папке `TeloPon/themes/`.

```text
TeloPon/
 └── themes/
      ├── default.css        ← Стандартная тема
      └── my_custom.css      ← 🌟 Просто добавьте сюда свой CSS-файл!
```
※ Любой добавленный CSS-файл автоматически появится в выпадающем списке «⚙️ Тема телопа» в интерфейсе TeloPon.

---

## 🏗️ 2. [Важно] Структура HTML и два контейнера телопов

Дисплей TeloPon (`base.html`) имеет два встроенных контейнера телопов: **«Обычный контейнер»** и **«Специальный контейнер»**.
Какой контейнер используется, определяется автоматически по «ID окна» (описано ниже), который задаёт ИИ.

### ① Обычный контейнер телопа (normal group)
Отображает несколько телопов в формате «каскада» (смещённые/наложенные), как правило, в верхнем правом углу экрана. Каждый раз, когда появляется телоп, JavaScript динамически создаёт элемент `.telop-item`.

```html
<!-- Единственная «обёртка», существующая на странице -->
<div id="telop-multi-container">

    <!-- Каждый раз при появлении телопа JS автоматически генерирует такой элемент: -->
    <div class="telop-item window-variety layout-flat">

        <!-- Метка (бейдж) в верхнем левом углу телопа -->
        <div class="teloop-badge">AI</div>

        <!-- Текстовое содержимое -->
        <div class="text-wrapper">
            <div class="topic-line">Тема (верхняя строка)</div>
            <div class="main-line">Основной текст (нижняя строка)</div>
        </div>

    </div>

</div>
```

> **Примечание:** Элементы `.telop-item` динамически создаются JavaScript. Чтобы нацелить CSS на «все обычные телопы», пишите `#telop-multi-container .telop-item { ... }`.

### ② Специальный контейнер (special group)
Используется, когда нужно отобразить телоп «независимо (перекрывая)» с обычным телопом — например, пояснительные телопы внизу экрана или представления имён. Этот элемент существует как единственный фиксированный элемент на странице.

```html
<!-- Специальный контейнер (один фиксированный элемент на странице) -->
<div id="explain-container">

    <!-- Бейдж в верхнем левом углу специального контейнера (можно нацелить по ID) -->
    <div id="exp-badge"></div>

    <!-- Текстовое содержимое -->
    <div id="exp-text-wrapper">
        <div class="topic-line" id="exp-topic">Тема (верхняя строка)</div>
        <div class="main-line"  id="exp-main">Основной текст (нижняя строка)</div>
    </div>

</div>
```

## 🔍 3. Полный справочник селекторов для CSS

Здесь полный список «какой селектор нацеливается на какой элемент» при написании CSS темы.

### Обычный телоп (normal group) — Селекторы

| Селектор | Тип | Роль |
|---|---|---|
| `#telop-multi-container` | ID | Обёртка для всех обычных телопов. Напрямую стилизуется редко |
| `.telop-item` | Класс | Каждый отдельный элемент телопа. Сюда также добавляются ID окон и ID макетов |
| `.telop-item.visible` | Класс | Состояние, когда телоп отображается. Используется для управления анимацией |
| `.teloop-badge` | Класс | **Красная метка (бейдж)** в верхнем левом углу телопа. Стилизуйте это в CSS |
| `.teloop-badge.has-text` | Класс | Состояние, когда в бейдже есть текст (нужно показать). Показывается через `display: block` |
| `.no-badge` | Класс | Добавляется к `.telop-item`, когда нет бейджа. Используется для регулировки верхнего отступа |
| `.text-wrapper` | Класс | Обёртка, группирующая тему и основной текст |
| `.topic-line` | Класс | Тема (текст верхней строки) |
| `.main-line` | Класс | Основной текст (текст нижней строки) |
| `.h1` / `.h2` | Класс | Классы выделения, добавляемые к словам, которые ИИ хочет подчеркнуть. Используются для изменения цвета и т.д. |

#### Пример: Настройка бейджа

```css
/* Пример: переделка красного бейджа */
.teloop-badge {
    /* Когда бейдж скрыт (по умолчанию) */
    display: none;
    position: absolute;
    top: -45px;
    left: -55px;

    /* Визуальный дизайн */
    background: #ff0000;   /* Цвет фона (красный) */
    color: white;
    font-size: 26px;
    font-weight: 900;
    padding: 10px 15px;
    border-radius: 8px;
    border: 3px solid white;
    transform: rotate(-5deg);  /* Небольшой наклон */
    box-shadow: 4px 4px 10px rgba(0,0,0,0.5);
    white-space: nowrap;
    z-index: 10;
}

/* Когда в бейдже есть текст (показать его) */
.teloop-badge.has-text {
    display: block;
}
```

### Специальный контейнер (special group) — Селекторы

| Селектор | Тип | Роль |
|---|---|---|
| `#explain-container` | ID | Обёртка специального контейнера. Сюда добавляются классы ID окон |
| `#explain-container.visible` | ID + Класс | Состояние, когда специальный контейнер отображается. Используется для управления анимацией |
| `#exp-badge` | ID | **Бейдж в верхнем левом углу** специального контейнера (метка для окон пояснений) |
| `#exp-badge.has-text` | ID + Класс | Состояние, когда в бейдже есть текст. Показывается через `display: block` |
| `#exp-text-wrapper` | ID | Текстовая обёртка специального контейнера |
| `#exp-topic` | ID | Строка темы специального контейнера (также имеет класс `.topic-line`) |
| `#exp-main` | ID | Строка основного текста специального контейнера (также имеет класс `.main-line`) |

> **ID vs. Класс:** ID-селекторы (начинаются с `#`, например `#exp-badge`) нацеливаются на элементы, которые встречаются на странице только один раз. Классовые селекторы (начинаются с `.`, например `.teloop-badge`) могут применяться к нескольким элементам.

#### Пример: Настройка бейджа специального контейнера

```css
/* Пример дизайна бейджа для окна пояснений */
.window-explain #exp-badge {
    display: none;          /* Скрыт по умолчанию */
    position: absolute;
    top: -25px;
    left: 30px;
    background: #00d2ff;    /* Голубой */
    color: #000;
    font-weight: 900;
    padding: 5px 25px;
    border-radius: 20px;
    font-size: 24px;
    box-shadow: 0 4px 10px rgba(0,0,0,0.3);
}

.window-explain #exp-badge.has-text {
    display: block;         /* Показать при наличии текста */
}
```

### Контейнер изображений — Селекторы

| Селектор | Тип | Роль |
|---|---|---|
| `#img-container` | ID | Обёртка для отображения изображений. Сюда добавляются классы ID окон |
| `#img-container.visible` | ID + Класс | Состояние, когда изображение отображается |

## 🪪 4. Доступные ID окон и управление системой

TeloPon инструктирует ИИ использовать предопределённые «ID окон». В CSS вы пишете правила дизайна и системные команды, нацеленные на класс (`.window-xxx`) для каждого ID окна.

### 📌 ID обычных контейнеров (normal group)
Добавляются к `.telop-item` и исчезают по истечении времени, установленного в интерфейсе в разделе «Длительность обычного отображения (сек)».

* **`.window-simple`**: Для наиболее стандартных, простых телопов.
* **`.window-variety`**: Для броских дизайнов в стиле развлекательного шоу.
* **`.window-news`**: Для официальных дизайнов в стиле новостной трансляции.
* **`.window-cute`**: Для рукописных или милых/kawaii дизайнов.
* **`.window-digital`**: Для дизайнов в стиле киберпанк или цифровой эстетики.
* **`.window-reply`**: Для ответов зрителям или упоминаний (например, «Эй, @имя_пользователя!») — широкоформатные дизайны, привлекающие внимание.

### 📌 ID специальных контейнеров (special group)
Применяются к `#explain-container`. Чтобы сообщить системе «это специальный контейнер», необходимо **указать `--window-group: special;` в CSS**.

* **`.window-explain`**: Для длинных пояснений или дополнительной информации, отображаемых заметно внизу экрана.
* **`.window-nameplate`**: Для отображения «псевдонима + имени» в телевизионном стиле крупным текстом.

## 🔊 5. Воспроизведение звуков и управление системой (CSS-переменные)

Одна из главных особенностей TeloPon — возможность **«передавать настройки обратно в систему (на сторону Python/JS) из CSS»**. Это делается с помощью «CSS-переменных (деклараций, начинающихся с `--`)».

### 💡 Воспроизведение звуковых эффектов (`--sound-file`)
Чтобы воспроизвести звуковой эффект при появлении телопа, запишите `--sound-file` в CSS. Укажите аудиофайл из папки `TeloPon/sounds/`.

```css
/* Пример: воспроизводить звук "pop" при появлении окна variety */
.window-variety {
    background: linear-gradient(135deg, #ff00cc, #333399);
    color: white;
    /* Указывает системе воспроизвести TeloPon/sounds/sound1.mp3 */
    --sound-file: "sound1.mp3";
}
```

### 💡 Переключение контейнера в «специальный» режим (`--window-group`)
Как упоминалось выше, при создании дизайнов специальных контейнеров, таких как `window-explain`, пропуск этой переменной приведёт к тому, что они будут обработаны как обычный контейнер.

```css
.window-explain {
    position: absolute;
    bottom: 40px;
    /* Включение этого приводит к рендерингу в "#explain-container" */
    --window-group: special;
    --sound-file: "sound2.mp3";
}
```

### 💡 Принудительное переопределение длительности отображения (`--display-duration`)
Можно принудительно установить фиксированную длительность отображения для конкретного телопа, игнорируя секунды, настроенные в интерфейсе (например, 5 секунд).

```css
.window-nameplate {
    /* Переопределить настройку интерфейса — принудительно скрыть табличку с именем ровно через 5 секунд */
    --window-group: special;
    --display-duration: 5;
}
```

## 📐 6. ID макетов (настройка баланса текста)

В дополнение к ID окна, обычный телоп (`.telop-item`) также получает «ID макета». Используйте его для настройки **соотношения размеров темы и основного текста**.

* **`.layout-flat`**: Плоский макет — тема и основной текст одного размера.
* **`.layout-big-top`**: Крупный текст в верхней строке (тема), меньший в нижней (основной).
* **`.layout-big-bottom`**: Меньший текст в верхней строке (тема), крупный и заметный в нижней (основной).

**Пример использования CSS:**
```css
/* Когда задан flat, можно применять исключения, например убрать рамки для вида только текста */
.window-variety.layout-flat .topic-line {
    background: none !important;
    font-size: 36px !important;
}
```

## 🎬 7. Реализация анимаций (`.visible`)

Когда телоп появляется на экране, система добавляет класс **`.visible`** к контейнеру.
Используйте это вместе с CSS `transition` и `transform` для создания плавных анимаций появления.

### Анимация обычного телопа
Базовая анимация обычных телопов (`.telop-item`) — увеличение с затуханием — определена как фиксированный стиль в `base.html`. Чтобы переопределить её в CSS темы, используйте более конкретный селектор.

```css
/* Пример: изменить обычные телопы на "въезжает справа" */
#telop-multi-container .telop-item {
    opacity: 0;
    transform: translateX(80px);   /* Начать со смещения вправо */
    transition: opacity 0.4s, transform 0.4s;
}
#telop-multi-container .telop-item.visible {
    opacity: 1;
    transform: translateX(0);      /* Въехать на позицию */
}
```

### Анимация специального контейнера
Анимация специального контейнера (`#explain-container`) полностью управляется CSS темы. Установите `opacity` и `transform` на классе ID окна и изменяйте их при добавлении `.visible`.

```css
/* =========================================
   Пояснительный телоп (выезжает снизу вверх)
   ========================================= */
.window-explain {
    position: absolute; left: 40px; bottom: 40px;
    opacity: 0;                          /* Начальное состояние: прозрачный */
    transform: translateY(30px);         /* Начальное состояние: немного смещён вниз */
    transition: all 0.6s cubic-bezier(0.2, 0.8, 0.2, 1);
    --window-group: special;
}
.window-explain.visible {
    opacity: 1;                          /* Показан: полностью непрозрачный */
    transform: translateY(0);            /* Показан: перемещается на позицию */
}
```

## 💡 8. Советы по разработке (отладка)

Корректировка CSS темы путём каждый раз ожидания, пока ИИ заговорит, занимает много времени.
Когда нужно проверить и настроить дизайн, запустите TeloPon в режиме отладки (`-d`).

```bash
python telopon_live.py -d
```

В нижней части интерфейса появится **«Панель ручного управления»**.
Введите тему, основной текст и ID окна, которое хотите протестировать, затем нажмите кнопку «Принудительное отображение», чтобы запускать тесты рендеринга (включая подтверждение воспроизведения звука) сколько угодно раз — без ИИ.
Доводите дизайн до совершенства, наблюдая в реальном времени за применением изменений CSS в браузерном источнике OBS!
