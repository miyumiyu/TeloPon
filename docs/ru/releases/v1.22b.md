# 🎛️ Release V1.22b: Новый режим «Авто-сегмент» и обновлённый интерфейс!

Главное в этом обновлении — новая функция **«Авто-сегмент»**.
Также переработан интерфейс выбора режима речи для более удобного использования.

---

## ✨ Новое и улучшенное

### 🆕 Новая функция: Режим «Авто-сегмент»

Когда с момента последнего телопа проходит заданное время, ИИ автоматически выводит телоп с резюме и структурированием обсуждённых тем.

* Выберите **«Авто-сегмент»** в выпадающем списке «Режим речи» в интерфейсе для активации
* Установите интервал срабатывания с помощью ползунка «Время вмешательства (секунды)»
* Рядом с выпадающим списком отображается **обратный отсчёт** до следующего срабатывания в реальном времени

**Идеально подходит для:**
Новостных/информационных стримов или долгих разговорных сессий, где периодические резюме от ИИ помогают зрителям следить за ходом беседы. Рекомендуемый интервал — 60–120 секунд.

---

### 🔄 Улучшение интерфейса: Выбор режима речи через выпадающий список

Прежний флажок «Авто-речь ВКЛ/ВЫКЛ» заменён на **выпадающий список для выбора режима речи**.

| Режим | Поведение |
|---|---|
| **None** | Без автоматического вмешательства |
| **Авто-речь** | ИИ проактивно предлагает тему через N секунд тишины (существующая функция) |
| **Авто-сегмент** | ИИ резюмирует и структурирует темы через N секунд после последнего телопа (новая функция) |

* Обратный отсчёт рядом со списком показывает **секунды до следующего срабатывания** с первого взгляда
* Отсчёт не останавливается, пока ИИ думает — точное время срабатывания всегда под контролем
* Режим можно переключать даже во время прямого эфира

### 🔧 Улучшение функций

* Плагины YouTube и Twitch теперь можно подключать и отключать после начала прямого эфира

### 🐛 Исправления и стабильность

* Исправлена проблема, при которой кратковременные шумы (кашель, посторонние звуки) неверно распознавались как речь, вызывая частые «пустые ходы»
  (Теперь звук должен длиться не менее 0,3 секунды, чтобы считаться речью)
* Исправлена проблема, при которой ИИ ошибочно воспринимал стримера как молчащего во время речи
* Исправлена проблема, при которой соединение с API не восстанавливалось корректно после разрыва со стороны сервера
