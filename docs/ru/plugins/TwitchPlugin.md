# 🎮 Инструмент интеграции с Twitch (TwitchPlugin.py)

Этот плагин позволяет просто указать имя канала или URL Twitch, чтобы **подхватывать комментарии зрителей в режиме реального времени и заставлять ИИ реагировать на них**.

Кроме того, если вы укажете **Client ID и Client Secret Twitch** (необязательно), то «название», «категория» и «изображение-превью» трансляции также будут автоматически получены и отправлены ИИ. **Плагин работает для чтения чата даже без Client ID.**

---

## ⚙️ Использование

### 1. Откройте панель настроек

Нажмите кнопку **«⚙️ Настройки»** для **«Twitch Live»** на панели «Расширения (плагины)» в правой части главного экрана TeloPon.

### 2. Введите имя канала

Введите имя канала Twitch (например, `mychannel`) или URL (например, `https://twitch.tv/mychannel`) в поле ввода вверху.

### 3. Укажите Client ID / Client Secret (необязательно)

Если вы также хотите поделиться с ИИ названием трансляции, категорией и превью, введите **«🔑 Client ID»** и **«🔒 Client Secret»**.

> ⚠️ **Чтение чата работает без этих данных.** Вы можете пропустить этот шаг, если вам нужна только интеграция чата.

**Как получить Client ID / Client Secret:**

1. **Включите двухфакторную аутентификацию (2FA)** на вашем аккаунте Twitch (обязательно).
   Без 2FA Twitch не позволит перейти к экрану регистрации приложения.
   👉 Настройка: [https://www.twitch.tv/settings/security](https://www.twitch.tv/settings/security) → «Настроить двухфакторную аутентификацию»

2. Войдите на [https://dev.twitch.tv/console](https://dev.twitch.tv/console) с вашего аккаунта Twitch.
3. Нажмите **«Зарегистрировать приложение»** и заполните следующие поля:

   | Поле | Пример | Примечание |
   |---|---|---|
   | Название | `TeloPon` | Любое удобное название |
   | OAuth Redirect URL | `http://localhost` | **Подойдёт фиктивный URL** (причина ниже) |
   | Категория | `Other` | Любая категория |
   | Тип клиента | **Confidential** | Обязательно для серверной аутентификации |

   > **💡 Почему нужно вводить `http://localhost`?**
   > TeloPon обращается к Twitch API через «Client Credentials Grant» — **серверный** метод аутентификации, при котором не требуется никакого экрана входа пользователя и фактически никакого редиректа. Redirect URL не используется. Однако форма регистрации Twitch требует заполнить это поле, поэтому ввод `http://localhost` в качестве фиктивного значения является стандартным и общепринятым решением.

4. После регистрации нажмите **«Новый секрет»** на странице приложения, чтобы сгенерировать Client Secret.
5. Скопируйте «Client ID» и «Client Secret» и вставьте их в панель настроек TeloPon.

### 4. Нажмите кнопку «Подключить»

После ввода данных нажмите кнопку **«Подключить»**.

* **Без Client ID**: появится «✅ Чат подключён», и чтение чата станет активным.
* **С Client ID**: информация о трансляции и превью будут получены и отображены, появится «✅ Подключено».

Убедитесь, что значок плагина стал зелёным `ON`, затем закройте панель.

### 5. Запустите подключение к эфиру

Вернитесь на главный экран TeloPon и нажмите **«🔴 Начать эфир»**, чтобы начать сеанс с ИИ.

Интеграция завершена! Как только сеанс ИИ начнётся, информация Twitch будет передана ИИ, и он будет автоматически реагировать на комментарии зрителей.

---

## 🧠 Информация, передаваемая ИИ

| Информация | Требование | Время |
|---|---|---|
| Название и категория трансляции | Требуется Client ID | При старте эфира |
| Изображение-превью | Требуется Client ID | ~5 секунд после старта |
| Комментарии зрителей | Всегда (анонимный IRC) | Реальное время (пакетами каждые 5 сек.) |

### Поведение при отправке комментариев

* Комментарии чата отправляются ИИ пакетами каждые **5 секунд** или при накоплении **10 комментариев**.
* Каждый комментарий доставляется ИИ в формате: `[COMMENT] имя_пользователя: текст комментария`

---

## ⚠️ Примечания

* **Порядок действий**
  Чтобы ИИ правильно распознал название и категорию трансляции, настоятельно рекомендуется **«Подключиться» в этом инструменте до нажатия кнопки «🔴 Начать эфир»**. (При подключении в середине сеанса название и категория не будут распознаны.)

* **Оффлайн-каналы**
  В режиме с Client ID, если целевой канал не в эфире, информация о трансляции не может быть получена. Подключение к чату при этом продолжает работать.

* **Слишком много комментариев**
  Для крупных трансляций с большим потоком комментариев ИИ может попытаться реагировать на каждый и говорить без остановки.
