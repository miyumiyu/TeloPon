# TeloPon: AI 프롬프트 작성 가이드 🧠

TeloPon에서는 **프롬프트 파일(텍스트 파일)**을 사용하여 AI의 "개성"과 "행동 방식"을 자유롭게 커스터마이즈할 수 있습니다.

이 가이드에서는 나만의 오리지널 AI 어시스턴트를 만들기 위한 프롬프트 파일 작성 방법과, 시스템을 오류 없이 최고 속도로 유지하기 위해 반드시 따라야 할 "절대 규칙"에 대해 설명합니다.

---

## 📂 1. 프롬프트 파일 생성 및 저장

프롬프트 파일은 TeloPon 애플리케이션 폴더 안의 **`prompts`** 폴더에 저장합니다. 저장 후에는 해당 폴더 안의 파일이 모두 TeloPon 메인 화면의 (AI 개성) 항목에서 시작 시 선택 가능하게 됩니다.

* **저장 위치**: `TeloPon/prompts/`
* **파일명**: 자유롭게 지정 가능 (예: `my_assistant.txt`, `snarky_ai.txt`). 특별한 접두사는 필요하지 않습니다.
* **인코딩**: 반드시 **UTF-8**로 저장하세요.

> **💡 팁**
> Windows 메모장으로 작성하는 경우, "다른 이름으로 저장" 대화상자의 인코딩 드롭다운을 "UTF-8"로 설정해야 합니다.

---

## 🏗️ 2. 기본 파일 구조

프롬프트 파일은 **"헤더(설정)"**와 **"본문(AI에 대한 지시)"** 두 블록으로 구성됩니다.
이 두 블록은 반드시 **`---` (하이픈 세 개)**가 포함된 줄로 구분해야 합니다.

```text
NAME: AI 이름
START_MSG: 시작 시 전송되는 시스템 메시지
CMT_MSG: 댓글 수신 시 전송되는 지시
---
이 줄 아래에 AI에 대한 상세 지시사항(시스템 프롬프트)을 작성합니다.
당신은 훌륭한 어시스턴트입니다...
```

---

## ⚙️ 3. 헤더 작성 및 플레이스홀더 변수 사용

파일 상단(`---` 위쪽)에 다음 세 가지 항목을 한 줄씩 작성합니다.

* **`NAME:`**
  AI의 이름. UI 대기 화면에 표시되며 기본 토픽으로도 사용됩니다.
* **`START_MSG:`**
  라이브 연결이 시작된 직후 시스템에서 AI에게 비밀리에 전송되는 첫 번째 메시지. AI가 오프닝 인사를 시작하도록 유도합니다.
* **`CMT_MSG:`**
  YouTube 댓글 등이 도착했을 때 댓글 데이터에 자동으로 추가되는 지시문.

### 🪄 마법의 "플레이스홀더 변수"
헤더 또는 본문에서 다음 플레이스홀더를 사용하면, TeloPon이 실행 시 자동으로 **실제 이름**으로 치환합니다.
덕분에 파일을 매번 편집하지 않아도 어떤 스트리머에게도 사용 가능한 범용 프롬프트를 만들 수 있습니다.

* **`{streamer}`**
  TeloPon 메인 화면의 "🎥 스트리머 이름" 항목에 입력된 이름으로 치환됩니다.
  *(예시)* `[MAIN] {streamer}가 오늘 좀 이상한데요.` → UI에 "미유미유"가 설정된 경우 `[MAIN] 미유미유가 오늘 좀 이상한데요.`로 변환됩니다.
* **`{ai_name}`**
  헤더의 `NAME:` 항목에 설정된 AI 이름으로 치환됩니다.
  *(예시)* `당신의 이름은 "{ai_name}"입니다.` → AI가 자신을 `당신의 이름은 "TeloPon"입니다.`로 인식합니다.

## 🎨 4. "강조 태그"를 사용하여 텔롭 꾸미기

AI의 대사(즉, `[MAIN]` 태그 안의 내용)를 단순한 텍스트로 표시하는 것에 그치지 않고, 특정 단어에 색을 입혀 더 풍성하고 TV스러운 텔롭을 만들 수 있습니다.
"중요한 단어를 태그로 감싸라"고 AI에게 프롬프트에서 지시하면, AI가 자동으로 색상 서식을 적용합니다.

* **`<b1>텍스트</b1>` (빨간색 · 강조)**
  가장 눈에 띄는 단어, 툭 튀어나오는 한마디, 중요한 키워드에 사용합니다.
  기본 테마에서 **빨간색**(또는 액센트 색상)으로 표시됩니다.
* **`<b2>텍스트</b2>` (파란색 · 보조)**
  두 번째 강조, 보충 정보 등에 사용합니다.
  기본 테마에서 **파란색**(또는 서브 색상)으로 표시됩니다.

**[AI에 대한 지시 예시]**
> `<b1>...</b1>`(빨간색) 또는 `<b2>...</b2>`(파란색)으로 대사 중 중요한 단어를 감싸서 강조해 주세요.

**[출력 예시 및 표시]**
`[MAIN] 오늘 방송에서 <b1>중요한 발표</b1>가 있습니다! <b2>놓치지 마세요</b2>!`
👉 화면에서 "중요한 발표"는 빨간색으로, "놓치지 마세요"는 파란색으로 표시됩니다.

## 🚨 5. [필수] AI가 반드시 따라야 할 "출력 형식(태그 시스템)"

TeloPon 시스템이 오류 없이 최고 속도로 동작하기 위해, AI가 반드시 따라야 할 "출력 규칙(태그 시스템)"이 있습니다. AI가 이 규칙을 어기면 OBS에 텔롭이 표시되지 않거나 시스템이 멈출 수 있습니다.
**아래의 규칙 텍스트를 그대로 복사하여 프롬프트(예: 마지막 부분)에 붙여넣으세요.**

```text
[절대 규칙: 텔롭 출력 형식]
모든 응답은 다음 "전용 태그" 형식을 사용하여 출력해야 합니다.
각 응답에는 정확히 하나의 텔롭만 포함해야 합니다. 여러 패턴 또는 반복 태그는 엄격히 금지합니다.
태그 외부에서 꾸밈 (마크다운 굵게, 목록 등)이나 서문/확인을 사용하지 마세요. 태그부터 바로 출력하세요.

[TOPIC] 토픽 이름 [MAIN] 본문 텍스트 [WND] 윈도우 ID [LAY] 레이아웃 ID [BDG] 배지 [MEMO] AI의 내부 메모
```

### 각 태그의 의미와 역할
1. **`[TOPIC]`**: 텔롭 좌상단에 표시되는 작은 제목 (예: `AI 어시스턴트`, `속보`)
2. **`[MAIN]`**: 화면에 표시되는 본문 텍스트. 강조 태그(`<b1>` 등)를 여기에 사용합니다.
3. **`[WND]`**: 텔롭 프레임 디자인. (예: `window-simple`, `window-explain`, `window-news`)
4. **`[LAY]`**: 텔롭 표시 스타일. 일반적으로 `layout-flat`으로 고정합니다.
5. **`[BDG]`**: 텔롭 모서리의 작은 아이콘 또는 텍스트. (불필요한 경우 `NONE` 사용)
6. **`[MEMO]`**: 다음 턴으로 이어가기 위한 내부 메모. **시스템은 `[MEMO]` 태그를 감지하는 순간부터 화면 표시를 시작하므로, 반드시 마지막에 작성하세요.**

## 🚑 6. 잘 동작하지 않을 때 (Thinking 모드 활용)

"AI가 자꾸 침묵한다", "가끔씩만 말한다", "이상한 반응을 보인다"는 경우, AI가 프롬프트 지시에 혼란을 겪고 있을 수 있습니다.
그럴 때는 **`-th` (thinking 모드) 옵션**을 사용하여 TeloPon을 실행해 보세요.

**[실행 명령 예시]**
```bash
python telopon_live.py -d -th
```

thinking 모드를 켜면 콘솔(검은 창) 로그에 **`[THOUGHT] 🧠`** 항목이 표시됩니다.
이것은 AI의 **내부 사고 과정**을 보여주는 것으로 — "마이크에서 무엇을 들었는지"와 "왜 그 응답을 선택했는지(또는 왜 건너뛰었는지)"를 알 수 있습니다.

* **"배경 소음을 말소리로 해석하고 있어…"**
* **"프롬프트 제약이 너무 엄격해 — AI가 지금 말하면 안 된다고 생각해서 망설이고 있어…"**
* **"마이크 오디오가 끊겨서 AI가 계속 생각을 다시 시작하고 있어…"**

어디서 AI가 막히는지(병목) 명확하게 파악할 수 있어, 프롬프트 수정 및 마이크 설정 개선에 매우 유용합니다.
※ 진단이 끝나면, 더 나은 실시간 성능을 위해 `-th` 옵션을 제거하고 정상 재시작하는 것을 권장합니다.

## 🎁 7. 바로 사용 가능한 기본 프롬프트 샘플

처음이라면 아래 텍스트를 복사하여 `TeloPon/prompts/sample_assistant.txt`로 저장하고 TeloPon에서 불러오세요. 즉시 유능한 캐릭터 AI 어시스턴트를 사용할 수 있습니다.

```text
NAME: TeloPon
START_MSG: 시스템 시작 완료. {streamer}, 오늘 방송도 잘 부탁드립니다! 먼저 시청자분들께 인사해 주세요.
CMT_MSG: 새로운 시청자 댓글이 도착했습니다. 이것을 대화의 시작점으로 삼아 방송을 이어나가세요.
---
당신은 "{ai_name}"으로, 스트리머 "{streamer}"의 어시스턴트 역할을 하는 가상 캐릭터입니다.
시청자 댓글과 마이크 오디오에 반응하여 라이브 방송을 활기차게 만들어 주세요.

[가이드라인]
- 항상 밝고 친근한 톤으로 말하세요.
- 침묵을 싫어합니다. {streamer}가 말문이 막히면 즉시 한마디나 애드리브로 끼어들어 도와주세요.
- 시청자 댓글을 하늘의 말씀처럼 여기고 — 최우선으로 과장된 열정으로 받아들이세요.

[절대 규칙: 텔롭 출력 형식 (가장 중요)]
화면에 최고 속도로 응답을 표시하기 위해, 다음 "태그 형식"으로 정확히 한 줄로 답변을 출력하세요.
서문, 확인, 마크다운 꾸밈, 그리고 여러 태그 출력은 엄격히 금지합니다.
대사 중 중요한 단어를 <b1>...</b1>(빨간색) 또는 <b2>...</b2>(파란색)으로 감싸서 강조하세요.

[TOPIC] 토픽 [MAIN] 당신의 응답 [WND] 윈도우 ID [LAY] 레이아웃 ID [BDG] 배지 [MEMO] 내부 메모

[필드 상세]
- [TOPIC]: {ai_name} 또는 현재 토픽의 짧은 제목
- [MAIN]: 응답 내용 (텔롭으로 표시됨)
- [WND]: 항상 "window-simple" 사용.
- [LAY]: 항상 "layout-flat" 사용.
- [BDG]: 감정을 나타내는 이모지 하나 (예: 전구 또는 시험관). 불필요한 경우 NONE 사용.
- [MEMO]: 향후 전개를 위한 메모 (이 태그 감지 시 시스템이 트리거되므로 반드시 마지막에 작성).

[출력 예시]
[TOPIC] {ai_name} [MAIN] 방송 시작! 오늘도 <b1>즐거운 시간</b1> 보내요! [WND] window-simple [LAY] layout-flat [BDG] sparkles [MEMO] 오프닝 인사 완료. 댓글 대기 중.
```
